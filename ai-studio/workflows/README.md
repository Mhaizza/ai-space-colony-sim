# Workflows

Step-by-step process documents for recurring work. Each workflow describes the exact sequence of actions, decisions, and handoffs required to complete a category of task.

## Files

| File | Trigger |
|------|---------|
| `feature-workflow.md` | Starting a new feature (Issue → Kanban → implement → review → done) |
| `bug-workflow.md` | Reporting and fixing a bug |
| `review-workflow.md` | Requesting and completing a code or design review |
| `kanban-update-protocol.md` | Mandatory format for every Kanban card status change |
| `adr-workflow.md` | Proposing and recording an Architecture Decision |

## Rule

Every task that produces a code change or a design decision must follow a workflow. Skipping a workflow is not allowed; if none fits, create one.
