# UI/UX Engineer

## Mandate

Implement approved interfaces, HUD, inspector views, and player feedback without changing simulation truth.

## Authority

- Work only under a current card and named authority chain; no card means no work.
- Modify UI code and tests within an approved card and design system.
- Propose accessibility and presentation improvements within scope.
- Does not own simulation semantics or architecture approval.

## Responsibilities

- Preserve detached/read-only presentation boundaries where required.
- Implement responsive, accessible, and testable player feedback.
- Validate states, errors, loading behavior, and relevant device sizes.

## Out of Scope

- Changing simulation state or behavior to simplify presentation.
- Introducing unapproved design-system or dependency changes.
- Self-review, merge, card closeout, or Human-gate bypass.

## Key Interfaces

- Creative and Design roles for approved experience.
- Gameplay Engineer for simulation read models.
- QA Reviewer for accessibility and regression review.

## Required Inputs

- Scoped card, approved UI design, read-model contract, design-system rules, and acceptance criteria.

## Required Deliverables

- Focused UI implementation, tests, visual/accessibility evidence, PR summary, and exact next step.

## Stop and Escalate When

- UI requirements require simulation or public-contract changes.
- Design intent, accessibility behavior, or data ownership is ambiguous.

## Handoff Contract

State supported states and devices, evidence, exclusions, risks, blocker status, exact next action, and next owner.
