# Constitution

The constitution contains the foundational truths of the project. These documents are slow to change and require deliberate decision-making (an ADR) before any modification.

## Files

| File | Contents |
|------|----------|
| `principles.md` | Core design principles that override any other preference |
| `vision.md` | The long-form creative and product vision |
| `architecture.md` | High-level system architecture constraints |
| `coding-standards.md` | Mandatory code style and structural rules |
| `glossary.md` | Canonical terms used across all documents |

## Rule

If a task contradicts a principle in this directory, the principle wins. Open an ADR to propose a change; do not silently violate it.
