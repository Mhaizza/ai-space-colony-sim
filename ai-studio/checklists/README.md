# Checklists

Quality gates that must be cleared at defined points in a task lifecycle. Checklists are not guidelines — every item is a hard requirement.

## Files

| File | When to Use |
|------|-------------|
| `task-start.md` | Before writing any code or design |
| `pre-commit.md` | Before every `git commit` |
| `pre-review.md` | Before requesting a review |
| `phase-complete.md` | Before marking a Kanban phase as done |

## Rule

Do not skip checklist items. If an item is genuinely not applicable, note it explicitly (`N/A — reason`) rather than leaving it blank.
